### **SYSTEM PROMPT / DATABASE CONTEXT**

Here is the structure of my database. Please use this schema for any SQL queries or logic I ask for.

#### **PART 1: Conceptual ER Diagram (From Picture 1.1)**

This is a conceptual ERD describing Restaurants, Employees, and Shifts.

**Entities & Attributes:**

1. **Restaurants**
* `restaurantID` (int) - **Primary Key**
* `name` (nvarchar(100))
* `address` (Composite attribute consisting of: `street` nvarchar(50) and `city` nvarchar(50))


2. **Employees**
* `empID` (int) - **Primary Key**
* `FullName` (nvarchar(50))
* `gender` (char(1))
* `phone` (nvarchar(20)) - **Multi-valued attribute** (represented by a double oval, meaning an employee can have multiple phone numbers).


3. **Shifts**
* `shiftID` (int) - **Primary Key**
* `shiftDate` (date)
* `startTime` (time)
* `endTime` (time)



**Relationships:**

* **has (1:N):** One `Restaurant` (1) has many `Employees` (N).
* **works (N:M):** Many `Employees` (N) work many `Shifts` (N). (This implies a bridging table is needed in physical implementation).

---

#### **PART 2: Physical Relational Database Schema (From 2nd Image)**

This represents an operational restaurant management database.

**Tables & Columns:**

1. **Tables**
* `TableID` (Primary Key)
* `TableNumber`
* `Capacity`
* `Location`


2. **Customers**
* `CustomerID` (Primary Key)
* `FullName`
* `Phone`
* `Email`
* `CreatedDate`


3. **Employees**
* `EmployeeID` (Primary Key)
* `FullName`
* `Role`
* `HireDate`
* `Salary`


4. **MenuItems**
* `ItemID` (Primary Key)
* `ItemName`
* `Category`
* `Price`
* `IsAvailable`


5. **Reservations**
* `ReservationID` (Primary Key)
* `CustomerID` (Foreign Key referencing `Customers.CustomerID`)
* `ReservationTime`
* `GuestCount`
* `Status`
* `Notes`


6. **ReservationTables** (Bridging table for Many-to-Many between Reservations and Tables)
* `ReservationID` (Primary Key, Foreign Key referencing `Reservations.ReservationID`)
* `TableID` (Primary Key, Foreign Key referencing `Tables.TableID`)


7. **Orders**
* `OrderID` (Primary Key)
* `CustomerID` (Foreign Key referencing `Customers.CustomerID`)
* `EmployeeID` (Foreign Key referencing `Employees.EmployeeID`)
* `ReservationID` (Foreign Key referencing `Reservations.ReservationID`)
* `OrderTime`
* `Status`
* `PaymentTime`
* `PaymentMethod`
* `TotalAmount`


8. **OrderTables** (Bridging table for Many-to-Many between Orders and Tables)
* `OrderID` (Primary Key, Foreign Key referencing `Orders.OrderID`)
* `TableID` (Primary Key, Foreign Key referencing `Tables.TableID`)


9. **OrderDetails** (Bridging table for Many-to-Many between Orders and MenuItems)
* `OrderID` (Primary Key, Foreign Key referencing `Orders.OrderID`)
* `ItemID` (Primary Key, Foreign Key referencing `MenuItems.ItemID`)
* `Quantity`
* `Price`
